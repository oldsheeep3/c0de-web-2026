# JavaScript講座（入門編） ── おみくじアプリを作ろう

**今回のゴール**：ボタンを押すと、運勢がランダムで表示される「おみくじ」を作る。
**テーマ**：JavaScriptのいちばん大事な流れ ──「**ボタンが押されたら、画面を書き換える**」を体験する。

> このアプリは**インターネット通信なし**（パソコンの中だけ）で動きます。
> 次のステップ「お天気アプリ（API編）」の**土台**になります。

---

## 0. 今日やること（全体像）

```
[ボタンを押す] → [運勢をランダムに1つ選ぶ] → [画面に表示]
  おみくじを引く      大吉・中吉・…から1つ         大吉
```

- ファイル：
  - `index.html` … ボタンと結果表示のわく
  - `omikuji.blank.js` … **穴埋め問題**（授業で埋める）
  - `omikuji.answer.js` … 模範解答（答え合わせ用）
  - `style.css` … 見た目

## 1. 変数 ── データに名前をつけて入れておく箱

```js
const result = document.getElementById("result");
//    ^名前              ^入れるもの
```

- `const`（コンスト）… 変数を作るときの合言葉。「これは箱ですよ」の印。
- `document.getElementById("result")` … HTMLの中から `id="result"` の要素を探して取り出す。
- これで、あとから `result` という名前でその要素を使える。

> **id とは？** HTMLの要素につける「名札」。`<div id="result">` のように付けておくと、
> JavaScriptから名指しで取り出せる。

---

## 2. 配列 ── たくさんのデータを順番に並べたもの

```js
const fortunes = ["大吉", "中吉", "小吉", "吉", "末吉", "凶"];
//                  0番     1番     2番    3番   4番    5番
```

- `[ ]` で囲み、`,` で区切って並べる。
- **番号は 0 から始まる**（1ではない！）。`fortunes[0]` は `"大吉"`。
- `fortunes.length` … 中身の個数（ここでは `6`）。

| 書き方 | 意味 | 結果 |
|---|---|---|
| `fortunes[0]` | 0番目 | `"大吉"` |
| `fortunes[5]` | 5番目 | `"凶"` |
| `fortunes.length` | 個数 | `6` |

---

## 3. ランダムに1つ選ぶ

「配列の中から、でたらめに1つ選ぶ」には2つの命令を組み合わせる。

```js
const index = Math.floor(Math.random() * fortunes.length);
```

分解して読むと：

| 順番 | 命令 | やること | 例 |
|---|---|---|---|
| ① | `Math.random()` | 0以上1未満の小数を作る | `0.73...` |
| ② | `× fortunes.length` | 0以上6未満に広げる | `4.38...` |
| ③ | `Math.floor(...)` | 小数点以下を切り捨てて整数に | `4` |

→ 結果、`index` には **0〜5 のどれか**が入る。
→ `fortunes[index]` で、その番号の運勢を取り出せる。

---

## 4. 関数 ── 「やることのまとまり」に名前をつける

```js
function draw() {
    // ここに処理を書く
}
```

- `function 名前() { ... }` … `{ }` の中の処理をひとまとめにして名前をつける。
- ここでは `draw`（引く）という名前にした。
- 名前を呼べば、中の処理がまとめて実行される。

> **ポイント**：関数は「作っただけ」では動かない。**呼び出して初めて**動く。
> 今回は「ボタンが押されたら呼び出す」ようにする（次の章）。

---

## 5. イベント ── 「〜されたら、〇〇する」

```js
drawButton.addEventListener("click", draw);
//  ^ボタン    ^イベントを登録     ^クリックされたら  ^draw を実行
```

- `addEventListener("click", 関数)` … 「クリックされたら、この関数を動かして」という予約。
- 関数名の後ろに **`()` を付けない**（`draw` であって `draw()` ではない）。
  - `draw()` と書くと「今すぐ1回実行」になってしまう。
  - 渡したいのは「押されたら動かす関数**そのもの**」。

---

## 6. 画面に表示する

```js
result.textContent = "大吉";
```

- `.textContent = "文字"` … その要素の中身の**文字**を書き換える。
- 「`=`」は「等しい」ではなく「**右のものを左に入れる**」という意味。

> `textContent`（文字だけ）と `innerHTML`（HTMLごと）の違いは、
> 次の天気アプリで `innerHTML` を使うときに詳しくやります。今回は `textContent` でOK。

---

## 7. 今日の完成コードの流れ（まとめ）

```
① 要素を取得   drawButton, result
② 配列を用意   fortunes = [大吉, 中吉, ...]
③ 関数 draw()  ├ ランダムな番号を作る    Math.floor(Math.random() * 個数)
               ├ 番号から運勢を取り出す  fortunes[番号]
               └ 画面に書き込む          result.textContent = 運勢
④ イベント登録 ボタンがクリックされたら draw を動かす
```

## 8. 演習の進め方

1. `index.html` をブラウザで開く（`omikuji.blank.js` が読み込まれている）。
2. `omikuji.blank.js` の **【 ① 】〜【 ④ 】** を埋める。
3. うまく動かないときは、**F12（開発者ツール）→ Console** でエラーを確認。
4. 完成したら、`omikuji.answer.js` と見比べて答え合わせ。

### 穴埋めのヒント早見表
| 番号 | やること | 答え |
|---|---|---|
| ① | ランダムな番号を作る | `Math.floor(Math.random() * fortunes.length)` |
| ② | 番号から運勢を取り出す | `fortunes[index]` |
| ③ | 画面に表示する文字 | `fortune` |
| ④ | クリックに反応させる | `addEventListener("click", draw)` |

## 9. 発展（余裕があれば）

- 運勢を増やす／減らす（配列に追加・削除するだけ）。
- 大吉のときだけ「おめでとう！」と一言そえる（`if` 文を使う）。
- ボタンを押すたびに背景色を変える。
- **次のステップ**：この「ボタン→関数→表示」の型のまま、
  運勢の代わりに**インターネットから天気を取ってくる**のが「お天気アプリ（API編）」。

## 用語まとめ

| 用語 | 意味 |
|---|---|
| **変数（const）** | データに名前をつけて入れておく箱 |
| **配列** | データを順番に並べたもの（番号は0から） |
| **関数** | やることのまとまりに名前をつけたもの |
| **Math.random()** | 0以上1未満のランダムな小数 |
| **Math.floor()** | 小数点以下の切り捨て（整数にする） |
| **イベント** | クリックなど「何かが起きたこと」 |
| **textContent** | 要素の中身の文字を書き換える |
