# JavaScript講座 ── お天気APIで天気を表示しよう

**今回のゴール**：都市名を入力すると、その街の「今の天気」を画面に表示するアプリを作る。
**テーマ**：**API（エーピーアイ）の練習** ── インターネット越しにデータをもらって、画面に出すまでの流れを体験する。

> ⚠️ **注意**：今回使うAPIは都市名を **ローマ字/英語** で受け取ります（`Tokyo`, `Osaka`, `London`）。
> 「東京」など日本語で入力すると見つからないので、授業では**ローマ字で入力**しよう。

---

## 0. 今日やること（全体像）

```
[都市名を入力] → [座標を取得する API] → [天気を取得する API] → [画面に表示]
    東京              緯度・経度            気温・天気コード         27℃ 晴れ
```

## 1. そもそも API とは？

**API = プログラムどうしがデータをやり取りするための「窓口」**。

身近なたとえ：**レストランの注文**

| レストラン | プログラム |
|---|---|
| メニューを見て注文する | 決められたURLにお願いする（リクエスト） |
| 店員さんが厨房に伝える | サーバーが処理する |
| 料理が運ばれてくる | データが返ってくる（レスポンス） |

- こちらは「厨房の中（どう作っているか）」を知らなくてよい。
- **決められた形でお願いすれば、決められた形でデータが返ってくる**。これがAPI。
- 今回は「天気を教えてくれるAPI」にお願いして、天気データをもらう。

### URLを叩いてみよう
ブラウザのアドレスバーに、次を貼り付けてみる（これがAPIへの「お願い」）：

```
https://api.open-meteo.com/v1/forecast?latitude=35.68&longitude=139.76&current=temperature_2m
```

→ 文字だらけのデータが返ってくる。これが **JSON**。

---

## 2. JSON（ジェイソン）とは？

**JSON = データを表すための書き方**。JavaScriptのオブジェクトとほぼ同じ形。

```json
{
  "current": {
    "temperature_2m": 27.3,
    "weather_code": 1,
    "wind_speed_10m": 3.4
  }
}
```

- `{ }` … オブジェクト（`キー: 値` の集まり）
- 値の取り出し方：`data.current.temperature_2m` → `27.3`
- `[ ]` … 配列（順番に並んだデータ）。`data.results[0]` で「最初の1個」

> ポイント：APIから返ってくるのは**ただの文字列**。
> `.json()` を使って、JavaScriptで扱えるオブジェクトに**変換**してあげる。

---

## 3. fetch ── APIにお願いする命令

`fetch(URL)` で、指定したURLにデータを取りに行く。

```js
const response = await fetch("https://.../forecast?...");  // ① お願いする
const data = await response.json();                        // ② JSONに変換
console.log(data.current.temperature_2m);                  // ③ 中身を取り出す
```

### `await`（アウェイト）って何？
通信は「返事が返ってくるまで時間がかかる」処理。
`await` を付けると、**「返事が返ってくるのを待ってから次に進む」**という意味になる。

- `await` は、`async` を付けた関数の中でしか使えない。

```js
async function getWeather() {   // ← async を付ける
    const response = await fetch(url);   // ← 待つ
    const data = await response.json();  // ← 待つ
    return data;
}
```

> **なぜ待つ？** 待たずに次へ進むと、まだ届いていないデータを使おうとして
> `undefined`（中身なし）になってしまうから。

## 4. 画面に表示する（DOM操作の復習）

取り出したデータを、HTMLに書き込む。

```js
const result = document.getElementById("result"); // 表示するわくを取得
result.innerHTML = "<p>27.3℃</p>";                // 中身のHTMLを書き換える
```

- `getElementById("id名")` … HTMLの要素を取り出す
- `.innerHTML = "..."` … その要素の中身を、指定したHTMLに差し替える

---

## 5. ボタンに反応させる（イベント）

「ボタンが押されたら、この関数を動かす」を登録する。

```js
searchButton.addEventListener("click", search);
//            ^イベントを登録   ^押されたら  ^search関数を実行
```

- `addEventListener("click", 関数)` … クリックされたら関数を呼ぶ
- 関数名の後ろに `()` は**付けない**（`search` であって `search()` ではない）
  - `()` を付けると「今すぐ実行」になってしまう。渡したいのは「関数そのもの」。

---

## 6. うまくいかない時のために（エラー処理）

通信は失敗することがある（電波が悪い、都市名が間違いなど）。
`try { } catch { }` で「失敗したときの動き」を用意しておく。

```js
try {
    // 成功したときの処理
    const place = await getLocation(city);
} catch (error) {
    // 失敗したときの処理
    result.innerHTML = "取得に失敗しました";
    console.error(error); // 開発者ツールに詳細を出す
}
```

## 7. 今日の完成コードの流れ（まとめ）

```
search()  ← ボタンが押されたら動く
  │
  ├─ getLocation(都市名)   … 都市名 → 緯度・経度   ［fetch①］
  │
  ├─ getWeather(緯度, 経度) … 座標 → 気温・天気     ［fetch②］
  │
  └─ showWeather(...)       … 画面に表示            ［innerHTML］
```

## 用語まとめ

| 用語 | 意味 |
|---|---|
| **API** | プログラム同士がデータをやり取りする窓口 |
| **リクエスト / レスポンス** | お願い / 返ってくるデータ |
| **JSON** | データの書き方（`{ キー: 値 }`） |
| **fetch** | URLにデータを取りに行く命令 |
| **async / await** | 「返事を待ってから進む」ための書き方 |
| **DOM** | HTMLをJavaScriptから操作するしくみ |
| **イベント** | クリックなど「何かが起きたこと」 |
